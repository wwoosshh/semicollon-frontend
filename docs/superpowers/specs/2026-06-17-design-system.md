# 세미콜론 플랫폼 — 디자인 시스템

동아리 메인 홈페이지의 디자인 언어를 플랫폼 전체에 일관 적용한다. 정체성: **에디토리얼 / 스위스 타이포그래피 / 개발자 zine** — 따뜻한 종이 배경, 단일 버밀리언 강조색, 모노스페이스 라벨, 얇은 그리드 선, 번호 매긴 섹션. **그림자·그라데이션·둥근 카드 금지**(= AI 슬롭 회피).

## 컬러 토큰 (CSS 변수, light 기준)

```
--paper:        #EFEDE7   /* 메인 배경 (따뜻한 크림) */
--paper-2:      #E8E4DB   /* 패널/hover 배경 */
--ink:          #1A1A18   /* 본문/제목 (near-black) */
--ink-2:        #56524B   /* 보조 텍스트 */
--muted:        #8A857C   /* 모노 라벨/3차 텍스트 */
--line:         #D6D1C6   /* 헤어라인 보더(그리드 선) */
--accent:       #DA3E22   /* 버밀리언 — ;, 번호, 링크, 상태점, active */
--accent-ink:   #B5301A   /* accent hover/pressed */
--dark:         #1B1916   /* 다크 밴드/푸터 배경 */
--on-dark:      #EDEAE2   /* 다크 위 텍스트 */
--on-dark-muted:#9A958B
```

강조색은 **아껴 쓴다** — 링크, 활성 상태, 상태점, `;`, 핵심 번호에만. 면적이 아니라 점·선으로.

## 타이포그래피

- **Display/제목 (한글)**: `Noto Serif KR` (700/900) — 페이지 타이틀, 큰 진술형 헤드라인.
- **Mono 라벨 (라틴/숫자/메타)**: `IBM Plex Mono` — **대문자 + letter-spacing 0.08~0.15em**. 섹션 번호, 필드 라벨(FIELD/STATUS), 테이블 헤더, 날짜, 코드성 메타. 가장 상징적.
- **본문/UI (한글+라틴)**: `Noto Sans KR` — 폼, 입력, 표 데이터, 설명문.
- 전부 `next/font/google`로 로드(Subset latin + korean). CDN 의존 없음.

## 형태/모티프

- **헤어라인 보더**(`1px var(--line)`)로 영역 구획. 박스 그림자 대신 선.
- **번호 섹션 헤더**: 빨강 모노 번호(`01`) + 한글 라벨 + 하단 가로 룰. 예: `01 우리가 하는 것`.
- **모노 마이크로 라벨**: `STATUS`, `FIELD`, `COHORT` 처럼 대문자 모노 + 자간 + `--muted`.
- **상태점**: `● OPEN`(accent) / `● 진행중` 같은 작은 원 + 모노 텍스트.
- **버튼**:
  - Primary: 배경 `--ink`, 텍스트 `--paper`, 각진(반경 2px), 라벨 뒤 ` →`. hover 시 `--accent` 배경.
  - Secondary: 투명 배경 + `1px var(--ink)` 보더. hover 시 `--paper-2`.
- **반경**: 2px(거의 각짐). 입력/버튼/뱃지 공통.
- **상단 얇은 빨강 바**(2~3px) 전역 최상단 — 브랜드 시그니처.
- **로고**: `세미콜론;` — `;`만 `--accent`.
- **마퀴/티커**(선택): 다크 밴드에서 가로 흐름 텍스트로 포인트.

## 앱 적용 규칙

- **앱 셸**: 상단 네비(홈페이지와 동일 구조). 좌측 `세미콜론;` 로고, 중앙 번호 메뉴(`01 홈 · 02 멤버 · …`), 우측 사용자명 + 아웃라인 `로그아웃` 버튼. 최상단 빨강 바. (현재의 좌측 사이드바 → 상단 네비로 교체.)
- **페이지 타이틀**: `섹션번호(빨강 모노) + 한글 제목(serif)` + 하단 헤어라인.
- **테이블(멤버 등)**: 헤더는 모노 대문자(`이름 / 역할 / 기수`는 한글이라 sans 라벨 + 보조 영문 모노), 행은 헤어라인 구분. 역할 `운영진`은 accent 텍스트/뱃지.
- **폼(로그인/가입)**: 에디토리얼 — 좌측 진술형 헤드라인(serif) + 우측 폼(모노 필드 라벨 + 각진 입력 + 검정 primary 버튼 `로그인 →`). 모바일은 세로 스택.
- **카드(Space/프로젝트)**: 홈페이지의 스터디/프로젝트/행사 카드처럼 — 번호 + 제목(serif) + 영문 라벨(mono) + 설명, 헤어라인 구획, hover 시 accent 디테일.

## 접근성·반응형

- 본문 대비 충분(ink on paper). accent는 텍스트로 쓸 때 충분한 크기/대비 확보.
- 모바일: 상단 네비는 햄버거 또는 축약, 그리드는 1열로 무너뜨림. 데스크톱은 그리드 유지.
